PIXS SYSTEM FLOW, CRUD, AND DATABASE DOCUMENTATION
==========================================================

1. PRODUCT DETAIL PAGE (Viewing and Adding to Cart)
--------------------------------------------------
ANG FLOW:
- Dito nag-sisimula ang user. Pinipili nila ang Variant (Size), Colors (Sequence), Screenplate (Availability), at Quantity.
- Kapag kumpleto na ang parameters, pwede i-click ang "Add to Cart" para ma-save sa cart_items table or "Buy Now" para diretso sa Transactions.

MGA FILLUPAN (FRONTEND):
- Variant Selection: Pagpili ng size/dimensions ng product.
- Color Sequence: Pagpili ng mga kulay (Primary, Secondary, Accent) depende sa channels ng screenplate.
- Quantity Picker: Bilang ng order (dapat hindi bababa sa minimum order).
- Screenplate Selector: Pagpili ng compatible na screenplate ng customer.
- Custom Requirements: Text area para sa mga kailangang instructions (logo placement, etc).

CRUD OPERATIONS:
- READ: Kinukuha ang data ng Product (variants, colors, gallery) at compatible Screenplates ng user.
- CREATE: Nag-gegenerate ng bagong 'cart_items' at 'cart_item_colors' sa database kapag clinick ang "Add to Cart".

MGA TABLES NA GAMIT:
- products: Main product info.
- product_variants: Sizes and prices.
- colors: List of master colors available.
- screenplates: Compatible screenplates owned by the user.
- screenplate_compatibility: Validation kung pwede yung screenplate sa product.
- product_gallery: Images shown in the slider.
- cart_items: Dito pini-persist yung selection.
- cart_item_colors: Dito pini-persist yung napiling colors for that cart item.

BACKEND / CONTROLLERS:
- ProductController@show: Para Makita yung product detail.
- CartController@store: Ginagamit itong endpoint para mag-save o mag-update ng item sa cart.


2. ADD TO CART PAGE (Cart Review & Management)
---------------------------------------------
ANG FLOW:
- Ito yung checkout review node. Dito pwedeng baguhin uli yung quantity, variant, or colors bago mag-checkout.
- May "Select All" functionality para mamili ng specific items na gustong bayaran.

MGA FILLUPAN (FRONTEND):
- Update Quantity: +/- buttons para baguhin ang dami.
- Update Variant: Pwede pang mag-palit ng size sa loob ng cart.
- Update Colors: Pwede palitan yung sequence ng kulay directly.
- Selection Toggle: Checkbox sequence para mamili ng items for checkout.

CRUD OPERATIONS:
- READ: Kinukuha lahat ng items sa 'cart_items' ng logged-in user.
- UPDATE: Ini-edit ang quantity, variant, o colors ng existing cart item.
- DELETE: "Remove" button para tanggalin ang item sa listahan.

MGA TABLES NA GAMIT:
- cart_items: Current stored items in cart.
- cart_item_colors: Relationship table para sa colors ng cart item.
- products & product_variants: Para sa computation ng subtotal at validation.

BACKEND / CONTROLLERS:
- CartController@index: Kinukuha listahan ng nasa cart.
- CartController@update: Pagbabago ng qty/colors.
- CartController@destroy: Pag-delete ng item.


3. TRANSACTIONS PAGE (Final Checkout & Order Creation)
-----------------------------------------------------
ANG FLOW:
- Ang final step bago maging Order. Dito nilalagay ang shipping at payment details.
- Pag-click ng "Purchase Now", lilinisin ang cart at gagawa ng official Order record.

MGA FILLUPAN (FRONTEND):
- Shipping Address: Pagpili sa pre-saved addresses ng customer.
- Payment Method: Pagpili sa card or method na gagamitin.
- Delivery Method: Pagpili sa courier options (Lalamove, J&T, etc).
- Production Notes: Final instructions for the manufacturing team.

CRUD OPERATIONS:
- READ: Kinukuha ang 'customer_addresses', 'customer_payment_methods', at 'delivery_methods'.
- CREATE: Gumagawa ng official entry sa 'orders', 'order_items', at 'order_item_colors'.
- DELETE (Cleanup): Tinatanggal yung mga binirang items sa 'cart_items' pagkatapos ng successful transaction.

MGA TABLES NA GAMIT:
- orders: Master order record (ID, status, total, customer_id).
- order_items: Snapshot ng product data (price, qty) nung binili ito.
- order_item_colors: Snapshot ng colors for that order item.
- customer_addresses: Shipping destination source.
- customer_payment_methods: Payment reference.
- delivery_methods: Delivery fees lookup.
- cart_items: (Cleanup phase) Tinatanggal dito ang mga successfully ordered items.

BACKEND / CONTROLLERS:
- OrderController@store: Ang main engine para sa transaction. Ginagawa nito atomically(DRY transaction) ang order entry at cart cleanup.
- DeliveryMethodController@index: Listahan ng couriers.
- CustomerAddressController@index: Listahan ng addresses ng user.
